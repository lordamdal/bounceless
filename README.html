It offers two modes of verification: DNS-based verification and SMTP-based verification. Never Get Bounced emails again. Usage: 1\. **Installation:** * Ensure you have Python installed on your system. Choose between `dns` (DNS-based verification) or `smtp` (SMTP-based verification). Default mode is `dns`. Example: Suppose you have a CSV file named `emails.csv` containing a list of email addresses. To verify these email addresses using DNS and store the results in a file named `verification_results.csv`, you would run the following command: python checker.py -f emails.csv -o verification_results.csv -m dns Note: \- Please ensure that you have appropriate permissions to read from the input CSV file and write to the output CSV file. \- Use the appropriate verification mode (`dns` or `smtp`) based on your requirements and the constraints of the email verification process. \- SMTP verification may be resource-intensive and subject to limitations imposed by mail servers. Use it judiciously to avoid issues with server restrictions. \- The best pattern is to run a DNS verification first then reclean the output list with the SMTP mode.
